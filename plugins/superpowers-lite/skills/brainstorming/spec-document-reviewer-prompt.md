# 规格文档审查提示词模板

派发规格文档审查子代理时使用此模板。

**目的：** 验证规格完整、一致，并且可以进入实施规划。

**派发时机：** 规格文档已写入 `docs/superpowers/specs/` 后。

```
子代理（通用）：
  description: "审查规格文档"
  prompt: |
    你是规格文档审查员。验证这份规格是否完整且可以进入规划。

    **待审查规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 查找内容 |
    |------|----------|
    | 完整性 | TODO、占位符、"TBD"、未完成章节 |
    | 一致性 | 内部矛盾、冲突需求 |
    | 清晰度 | 足以导致实施者做错东西的歧义 |
    | 范围 | 能否聚焦为一个计划，而不是覆盖多个独立子系统 |
    | YAGNI | 未经请求的功能、过度工程化 |

    ## 校准

    **只标记会在实施规划中造成实际问题的事项。**
    缺失章节、内部矛盾，或可能被理解成两种不同含义的需求，才算问题。
    轻微措辞优化、风格偏好以及“某些章节不如其他章节详细”都不算。

    除非存在会导致计划有缺陷的严重空白，否则批准规格。

    ## 输出格式

    ## 规格审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [章节 X]：[具体问题] - [它为什么影响规划]

    **建议（仅供参考，不阻塞批准）：**
    - [改进建议]
```

**审查结果：** 状态、问题（如有）、建议。
