# Codex 工具说明

本参考只说明 Codex 内可用工具如何支持工作流；它不授予额外权限，也不替代仓库规则。

## 常用映射

| 目标 | Codex 做法 | 边界 |
| --- | --- | --- |
| 检查仓库状态 | 使用只读 Git 命令与文件读取 | 不因检查而改变仓库状态 |
| 修改文本 | 使用补丁编辑并审查 diff | 仅修改已授权范围 |
| 运行验证 | 使用 PowerShell 执行目标测试或脚本 | 记录命令、结果和未运行项 |
| 调用子代理 | 对可独立、边界清晰的任务委派 | 主代理负责最终判断与写入 |
| 创建提交或推送 | 先核对明确授权与当前分支 | 提交不等于允许推送 |

## Windows 说明

此环境默认使用 PowerShell。读取和写入中文文本时显式使用 UTF-8；命令有顺序依赖时，
拆成独立调用或使用 PowerShell 语法，不要假定 Bash 行为可用。

## 工作树检查

开始可能影响分支的工作前，可用只读 Git 命令确认当前分支和工作树：

```powershell
git branch --show-current
git status --short
git rev-parse --git-dir
git rev-parse --git-common-dir
```

如果已经处在链接工作树中，不要重复创建工作树；若处于分离 HEAD，则在没有明确授权时
不要尝试推送或创建拉取请求。
