# 执行计划

行动前读取计划、稳定引用、仓库状态和进度证据。按**有意义的语义交付**推进，不按固定任务大小或仪式化待办清单机械执行。

## 保持计划有效

先确认计划仍符合当前代码、约束和用户意图。证据表明安全路径已经变化时，应**调整陈旧步骤**并记录原因，不能照抄已经被代码现状淘汰的操作。

只有遇到高影响阻塞或权限歧义才暂停：缺失决定会改变安全、架构、外部状态或语义验收。暂停只请求解除该阻塞所需的最小信息；不要自行扩大权限，也不要借机扩展范围。

## 推进交付

对每项交付：

1. 确认依赖和当前恢复检查点。
2. 完成边界清晰的修改。
3. 获取计划要求的语义证据并记录观察结果。
4. 更新计划或进度记录，使后续执行者无需依赖当前会话记忆。

只有 worktree、委派、评审或分支收尾在当前 Codex 环境和仓库策略下确实提高收益时，才使用对应工作流。不要假设必须公告、提交、推送或修改主分支。

收尾时报告已交付能力、架构关系、当前证据和剩余风险。真实的分支、集成、保留或丢弃决定必须进入仓库允许且已授权的流程。
