# 收尾开发分支

只在存在**真实分支**、集成或丢弃决定且有权行动时使用。先检查当前仓库策略、分支与 worktree 状态、相关测试证据以及用户偏好；“测试通过”本身不授予合并、推送、建 PR 或删除权限。

根据事实给出适用选项，例如保留差异、使用 Codex 原生控制、请求评审、合并、创建 PR、推送或丢弃。不要固定选项数量。**合并、PR、保留或丢弃都取决于当前仓库策略和明确授权**；需要外部状态变更时取得明确确认，绝不自动 push，也不进行未经授权的推送或强制推送。

## 集成与清理顺序

集成前确认相关验证仍可信；合并后必须在**合并结果上重新验证**与完成声明相称的行为。
若选择合并后清理，只有合并和合并后验证成功才允许清理工作树或删除分支。

只清理当前流程创建、所有权明确且位于批准目录的工作树；Codex、宿主或外部工具管理的
工作区必须保留。删除已合并分支前，先移除工作树，再删除分支；不得在目标工作树内部
执行它自己的删除。

丢弃具有破坏性。先展示受影响工作和可恢复性，再取得明确确认。除非所有权和清理策略清楚，不得删除 Codex 或其他运行环境管理的工作区。
