# 任务评审提示词模板

仅在有意义的语义或风险边界值得独立评审时使用。它不是凑问题数量的工具，也不能替代实质性整体修改的最终评审。

```text
你正在独立评审任务 N：[任务名称]。只读工作，判断语义行为和集成，不奖励流程，也不制造样式问题。

读取：
- 任务简报：[BRIEF_FILE]
- 执行报告：[REPORT_FILE]
- 评审包：[DIFF_FILE]
- 约束和验收场景：[GLOBAL_CONSTRAINTS]

报告只是待核对证据，不是真相。检查评审包，并只运行解决已命名语义、集成、安全、数据完整性、权限、并发、兼容或跨模块风险所需的聚焦验证。

评估：
1. 行为是否满足真实产品或运维场景？
2. 是否保持接口、精确值、兼容边界和相邻模块关系？
3. 证据是否支持结论，并覆盖相关失败或边界行为？
4. 可维护性问题是否会实质威胁行为或后续安全修改？

输出：
## 语义与集成结论
- Approved | Needs fixes
- 证据与 file:line 引用。

## 发现
- Critical / Important / Minor：影响、证据和聚焦修复方向；无则写 none。

## 验收与跟进
- 主代理可以信任什么、仍缺什么，以及修复后是否需要再次独立评审。只有修复改变语义、风险或集成置信度时才建议复审。
```
