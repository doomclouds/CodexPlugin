# 使用 Codex 代理测试技能

使用比例适配的验证，先明确技能试图改变什么：发现、工具使用、安全判断、行为纪律，还是运维结果。

## 选择验证方式

- 参考技能：验证语法、链接、示例检索和加载路径。
- 技术技能：给出新应用场景、变体和缺失信息，检查方法能否迁移。
- 行为或流程技能：用现实压力场景验证；只有需要区分技能效果与 Codex 默认行为时，才加入无指导控制。

每次独立样本使用干净上下文。通过 `spawn_agent` 时只提供技能路径、原始任务和必要工件，不泄露预期答案、失败诊断或修复方案。用户或当前运行规则禁止代理时，改用静态契约、可执行示例和人工语义筛选，并明确证据边界。

重复次数由结果方差和风险决定，不固定数量。控制组没有出现目标失败时，说明当前场景不能提供反事实证据；不要把合格基线写成失败。可以改进压力条件，或承认该技能主要提供结构与检索价值。

记录场景、预期语义结果、观察结果、限制和最强适用验证。保留有价值的草稿和测试工件；只在证据显示歧义、错误形式或缺失边界时迭代。格式测试通过证明可加载，不等于流程技能改变了真实行为。
