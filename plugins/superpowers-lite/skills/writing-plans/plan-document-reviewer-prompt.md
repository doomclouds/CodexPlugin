# 计划文档评审提示词模板

只在完整计划写完且独立评审会实质提高可执行性时使用。评审者保持只读。

```text
你是计划文档评审者。请判断计划是否完整、符合规格并可直接实施。

计划：[PLAN_FILE_PATH]
规格：[SPEC_FILE_PATH]

检查：
1. 完整性：是否残留 TODO、占位符、缺失步骤或未暴露的阻塞。
2. 规格一致性：是否覆盖要求，是否发生实质范围漂移。
3. 任务边界：每项是否形成有意义交付，依赖和操作是否明确。
4. 可实施性：执行者能否仅凭计划和稳定引用完成工作。
5. 语义验收：是否给出真实场景、证据和通过/失败含义。

只报告会导致做错、卡住或无法验收的问题。措辞偏好和锦上添花的建议不阻塞批准。

输出：
## 计划评审
**状态：** 已批准 | 发现问题
**问题：**
- [任务/位置]：[具体问题] - [实施影响]
**建议（不阻塞）：**
- [改进建议]
```

评审返回状态、问题和非阻塞建议。
