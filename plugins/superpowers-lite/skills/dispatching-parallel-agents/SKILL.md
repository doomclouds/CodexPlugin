# 并行委派代理

是否并行取决于任务独立性、协调成本、运行成本和 Codex 工作区安全。只有节省的关键路径时间与独立判断收益超过集成、冲突和恢复成本时才并行；拆分后需要大量对账的工作应顺序执行或合并处理。

委派前检查：

- 输入、输出、文件所有权和外部状态能否独立；
- 多个代理是否会修改共享可变目标；
- 接口、生成物或测试状态是否存在先后依赖；
- 并行后的净收益是否仍为正。

为每个 Codex 代理提供边界明确的任务、工作区、所有权、禁止项和预期证据。可用时通过 `spawn_agent` 启动独立任务，通过 `send_message` 补充非替代信息，通过 `wait_agent` 汇总状态；不要为了“用满并发”固定扇出数量。

代理返回后由主代理验证真实影响边界：共享接口、持久化状态、用户可见行为和集成点。各自绿色不等于集成通过。两个任务共享同一文件时，除非所有权能安全切分且协调成本仍低于收益，否则不要并行写入。
