# 技术探索

先写出本次必须回答的一个问题，再选择研究或原型。两者都只服务于决策，不偷偷演化成生产实现。

## 选择路径

### 研究

适用于外部事实：SDK、协议、标准、平台限制、产品能力和时效性较强的信息。

- 优先官方文档、标准、论文、源码和厂商资料。
- 对可能变化的事实进行实时核验，并把引用放在对应结论旁。
- 区分来源事实、工程推断和仍未证实的假设。
- 只有结果会被后续会话复用时，才按仓库惯例写入 `docs/research/` 或等价位置。

### 原型

适用于状态机、算法可行性、数据流水线或交互方向难以靠讨论决定的情况。

- 逻辑问题优先做最小可运行终端原型。
- UI 问题并列提供少量可切换方案，围绕同一决策比较。
- 原型必须有输入、观测指标和停止条件。
- 记录结论后丢弃原型代码；只有用户明确要求时才转为生产实现。

## 完成输出

输出明确答案、证据、推荐决策、已知限制和下一步。若证据不足，说明还缺哪一次实验或哪份一手资料。
